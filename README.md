# smart-news

AI-агрегатор новостей для экспатов. Читает Telegram-каналы-источники через MTProto,
фильтрует шум (мемы, рекламу, чернуху, дубли) через Claude и присылает годные новости
владельцу на модерацию — кнопки «Опубликовать / Пропустить». Одобренное уходит в публичный канал.

Первый регион — **Пхукет / Таиланд**, фид на русском.

```
каналы-источники ──MTProto──▶ ingest ──▶ Claude (фильтр+суммаризация) ──▶ черновик в личку ──▶ ✅ ──▶ канал
```

## Стек

- [Bun](https://bun.sh) + TypeScript
- [mtcute](https://mtcute.dev) — чтение каналов от имени личного аккаунта (MTProto)
- [grammY](https://grammy.dev) — бот модерации и публикации
- [OpenRouter](https://openrouter.ai) — LLM-доступ (классификация со structured outputs, рубрики с веб-поиском `:online`)
- SQLite (`bun:sqlite`) — состояние и очередь черновиков

## Запуск

1. **Ключи Telegram API**: [my.telegram.org](https://my.telegram.org) → API development tools → `api_id` + `api_hash`.
2. **Бот**: создай в [@BotFather](https://t.me/BotFather), возьми токен. Напиши боту `/start`, чтобы он мог писать тебе в личку.
3. **Канал**: создай канал для фида, добавь бота администратором с правом публикации.
4. **Свой user id**: узнай через [@userinfobot](https://t.me/userinfobot).
5. Конфиг:

   ```sh
   cp .env.example .env   # и заполни
   bun install
   ```

6. Вход в аккаунт (один раз, сессия сохраняется):

   ```sh
   bun run login
   ```

7. Запуск:

   ```sh
   bun start        # бот + опрос источников по расписанию
   bun run once     # разовый прогон без бота (для теста)
   ```

## Источники

Всё в [sources.json](sources.json), два типа:

- **`channels`** — Telegram-каналы (читаются через MTProto; публичные — по username
  без подписки).
- **`feeds`** — RSS англо- и тайскоязычных первоисточников: The Thaiger (лента
  Пхукета), Khaosod English, Bangkok Post, Khaosod (TH). Claude переводит и
  выжимает на русский — новости появляются в фиде раньше, чем их перепечатают
  русскоязычные каналы.

## Как работает отбор

1. **Триаж** — заголовки всех новых постов уходят одним дешёвым batch-запросом:
   очевидный шум отсеивается сразу, полная классификация только для кандидатов.
2. **Классификация** — полный текст кандидата: оставить/выкинуть, категория,
   важность, тональность, русский заголовок и выжимка.
3. **Сюжеты** — embedding русской выжимки сравнивается с недавними черновиками
   (72 ч): та же новость из другого источника не публикуется повторно, а
   вливается в существующий сюжет — в посте появляется список всех источников.
   Пограничные случаи подтверждает LLM.
4. **Модерация или автомат** — по умолчанию черновик приходит в личку с
   кнопками; с `AUTO_PUBLISH=1` всё прошедшее фильтр публикуется сразу, а в
   личку падает уведомление с кнопкой «🗑 Убрать из канала» (боты могут удалять
   посты младше 48 часов). Если автопубликация не удалась — черновик приходит
   на обычную модерацию, ничего не теряется.

Раз в неделю (или по команде `/scout`) **скаут** через веб-поиск ищет новые
каналы и RSS-фиды по региону, проверяет их живость и предлагает кандидатов
с кнопкой «Добавить» — список источников растёт без ручной возни.

## Регулятор негатива

`NEGATIVE_QUOTA_PCT` (по умолчанию 20) — максимум негативных постов в фиде, %.
Классификатор размечает тональность каждой новости; когда доля негатива за
последнюю неделю выше квоты, негативные посты отбрасываются. Исключение —
критичные предупреждения (важность 5: шторм, закрытый пляж, срочные изменения
правил) публикуются всегда.

## Ежедневные рубрики

Раз в день (после `RUBRIC_HOUR`, по умолчанию 10:00) канал получает не спарсенный,
а сгенерированный пост — Claude пишет его с живым веб-поиском:

- 🌴 **«Уголок Таиланда»** — место дня: что там делают туристы и местные (35 тем)
- 🤿 **«Чем заняться в Тае»** — активность дня с конкретикой для экспата (24 темы)

Рубрики чередуются, темы ротируются (сначала давно не использованные), список — в
[src/rubric-topics.ts](src/rubric-topics.ts). Рубрика проходит ту же модерацию,
что и новости.

## Команды бота

- `/check` — проверить источники прямо сейчас
- `/rubric [place|activity]` — сгенерировать рубрику вне расписания
- Кнопки под черновиком: ✅ опубликовать в канал, ❌ пропустить

## Настройка фильтра

Правила «что оставляем / что выкидываем» — системный промпт в
[src/classify.ts](src/classify.ts). Категории: safety, visa_docs, transport,
events, money, weather, infrastructure, other.

Модель по умолчанию — `anthropic/claude-opus-4.8` (через OpenRouter); для экономии
можно указать `LLM_MODEL=anthropic/claude-haiku-4.5` в `.env` — или любую другую
модель с поддержкой structured outputs из [каталога](https://openrouter.ai/models).
